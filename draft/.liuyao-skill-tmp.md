# Liuyao（六爻排盘解析）

用同目录下一个确定性的 Node 脚本解析六爻卦象，输出结构化结果：本卦/变卦、
每爻的六亲地支五行、世应、变爻、六神顺序，以及起卦日期的干支时辰。
纯本地计算，无网络、无凭证。

## 何时使用

- 用户要求 起卦 / 排盘 / 解卦，或提到 卦象、六爻、爻象、六亲、六神、世应。
- 出现 ` ```liuyao ` fenced 代码块，或用户直接给出卦象。

## 如何调用

用 exec_shell 运行解析器（脚本与本文档同目录）：

```sh
node "$HOME/.codewhale/skills/liuyao/parser.skill.js" "<卦象>" "<日期>"
```

- `<卦象>` 必填，三种形式（按此顺序尝试解析）：
  1. **四象数字串**（最精确）：6 个字符，每字符 ∈ `0123`，**从初爻（最左）到上爻（最右）**：

     | 字符 | 爻象 | 是否变爻 |
     |------|------|----------|
     | `0`  | 老阴 | 是 |
     | `1`  | 少阳 | 否 |
     | `2`  | 少阴 | 否 |
     | `3`  | 老阳 | 是 |

     例：`000111` → 初/二/三爻老阴、四/五/上爻少阳。
  2. **八卦名组合**：由 `乾坤震巽坎离艮兑` 组成的串。例：`坤坤震乾坎坎`。
  3. **卦名**：六十四卦名，如 `天风姤`、`天地否`、`乾为天`。
- `<日期>` 可选。任何 `new Date()` 能解析的串，如 `2026-07-13`、`2026-07-13T14:30`。
  省略时 `时辰信息` 为 `null`，但卦象照常返回。

## 输出结构

脚本用 `console.dir` 打印（单引号、含 `undefined`，是 JS 对象字面量而非严格 JSON——
直接读文本，不要 `JSON.parse`）：

```
{
  version,
  卦象: {
    本卦: {
      初爻/二爻/三爻/四爻/五爻/上爻: { 爻, 类型, 六亲地支五行 },  // 类型: '世' | '应' | null
      卦名, 宫, 变爻: [ '初爻', ... ]     // 无变爻时为 undefined
    },
    变卦: { ...同本卦结构... } | null,    // 无变爻时为 null
    世应变化: "..."
  },
  时辰信息: { 年, 月, 日, 时, 六神顺序: { 初爻..上爻 } } | null
}
```

注意：

- 有老阴(`0`)/老阳(`3`)才有变爻，此时 `变卦` 非空、`变爻` 列出变化之爻。
- 非世应爻的 `类型` 为 `null`。
- 缺 `<卦象>` → `卦象: undefined`；`<日期>` 无效 → `时辰信息: null`。

## 验证

运行后核对：打印的 `卦名` 与用户给出的形式一致；`变爻`/`变卦` 仅在存在老/变爻时出现。
本技能只输出排盘参考数据，不代替占断结论。
